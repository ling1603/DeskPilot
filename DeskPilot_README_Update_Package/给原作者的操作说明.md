# DeskPilot README 优化交付说明

你好，我对 DeskPilot 仓库的 README 做了一版专业化优化，目标是让项目在 GitHub 上更像一个可以被别人理解、安装、试用和收藏的开源项目。

## 本包包含什么

| 文件 | 用途 |
| --- | --- |
| `README_new.md` | 新版 README 内容，可直接替换仓库根目录的 `README.md` |
| `apply_readme_update.bat` | Windows 一键替换脚本，会自动备份旧 README |
| `apply_readme_update.sh` | macOS / Linux 一键替换脚本，会自动备份旧 README |
| `apply_readme_update.py` | 跨平台 Python 替换脚本 |
| `仓库改进建议.md` | README 之外建议顺手修复的问题 |
| `docs/images/README_images_placeholder.md` | 截图占位说明，后续可按说明补图 |

## 推荐操作方式

### Windows

1. 将本 ZIP 解压到 DeskPilot 仓库根目录，也就是和旧 `README.md` 同一级目录。
2. 双击运行：

```txt
apply_readme_update.bat
```

3. 脚本会自动：
   - 备份旧 README 为 `README.backup.时间戳.md`
   - 将 `README_new.md` 复制为新的 `README.md`

4. 打开 GitHub Desktop 或终端检查变更：

```bash
git diff README.md
```

5. 确认无误后提交：

```bash
git add README.md
git commit -m "docs: improve README"
git push
```

### macOS / Linux

```bash
chmod +x apply_readme_update.sh
./apply_readme_update.sh
```

### 通用 Python 方式

```bash
python apply_readme_update.py
```

## 建议同步处理

新版 README 中已经如实提示了当前源码可能存在的目录结构与 import 路径不一致问题。为了让用户真正能直接运行项目，建议进一步按 `仓库改进建议.md` 中的方案整理目录结构并补充 LICENSE、截图和图标资源。

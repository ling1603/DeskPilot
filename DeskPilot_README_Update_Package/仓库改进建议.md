# DeskPilot 仓库改进建议

以下内容不必全部写进 README，但建议原作者在维护仓库时逐步处理。

## 1. 优先修复目录结构与 import 路径不一致问题

当前代码中大量使用：

```python
from ui.main_window import MainWindow
from config.settings import ...
from services.organizer import FileOrganizer
from models.operation import OperationBatch
from utils.helpers import resource_path
```

但仓库页面中主要文件看起来都放在根目录，例如：

```txt
main.py
main_window.py
organizer.py
monitor_service.py
undo_manager.py
stats_service.py
tray_manager.py
settings.py
operation.py
rule_editor.py
helpers.py
```

建议整理为：

```txt
DeskPilot/
├── main.py
├── config/
│   ├── settings.py
│   ├── rules.json
│   ├── license.py
│   └── license.json
├── models/
│   └── operation.py
├── services/
│   ├── organizer.py
│   ├── monitor_service.py
│   ├── undo_manager.py
│   ├── stats_service.py
│   └── feature_gate.py
├── ui/
│   ├── main_window.py
│   └── rule_editor.py
└── utils/
    ├── helpers.py
    ├── tray_manager.py
    └── autostart.py
```

并在每个目录中添加空文件：

```txt
__init__.py
```

否则用户运行 `python main.py` 时可能会遇到：

```txt
ModuleNotFoundError: No module named 'ui'
ModuleNotFoundError: No module named 'config'
ModuleNotFoundError: No module named 'services'
```

## 2. 补充正式 LICENSE 文件

当前仓库中的 `license.py` 和 `license.json` 是 Free / Pro 功能门禁相关文件，不是开源许可证。

建议新增真正的 `LICENSE` 文件，例如：

- MIT License
- Apache License 2.0
- GPL-3.0 License

如果希望别人能自由学习、Fork 和二次开发，MIT 是比较常见、简单的选择。

## 3. 补充项目截图

建议新增目录：

```txt
docs/images/
```

推荐截图：

| 文件名 | 内容 |
| --- | --- |
| `main-window.png` | 主窗口整体界面 |
| `rules-editor.png` | 分类规则编辑器 |
| `tray-menu.png` | 系统托盘菜单 |
| `organize-result.png` | 整理前后文件夹效果 |

补图后，可以把 README 中的截图占位替换为真实图片。

## 4. 补齐图标资源

代码中尝试加载：

```txt
app.ico
app.png
```

建议统一整理到：

```txt
assets/app.ico
assets/app.png
```

同时修改代码中的资源路径为：

```python
resource_path("assets/app.ico")
resource_path("assets/app.png")
```

## 5. 明确 Python 版本

代码中使用了 `dict | None` 这种写法，建议 README 和 `pyproject.toml` / 文档中明确：

```txt
Python 3.10+
```

## 6. 增加打包说明

项目面向普通 Windows 用户时，建议补充 PyInstaller 打包脚本，例如：

```bash
pyinstaller --onefile --windowed --name DeskPilot main.py
```

并在 Release 中提供 `.exe` 文件，降低普通用户试用门槛。

## 7. 增加基础测试

可以补充针对核心整理逻辑的测试：

- 按扩展名分类是否正确；
- 未匹配扩展名是否跳过；
- 重名文件是否生成 `(1)`；
- 撤销是否能恢复文件；
- 规则文件损坏时是否能 fallback 到默认规则。

## 8. 优化撤销机制

当前撤销历史主要保存在内存中，程序重启后无法撤销上一轮整理。

后续可以考虑把操作历史持久化到 JSON 文件中，实现跨会话撤销。

## 9. 避免 README 夸大功能

Release 中提到“Desktop & Downloads folder monitoring”，但源码界面逻辑更像是“用户选择指定文件夹进行整理或监控”。

建议对外表述为：

> 可选择桌面、下载目录或任意文件夹进行整理；Pro 模式下可监控指定文件夹中新建文件。

这样更准确，也更不容易被用户误解。
